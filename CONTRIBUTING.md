## Contributing

Contributions are welcome! Whether you're reporting bugs, suggesting features, submitting pull requests, or improving documentation, I appreciate your involvement. This is a personal project I maintain for myself. If somebody else benefits from it, I'm glad.

### Submitting Issues

- Clearly describe the problem or feature request
- Include steps to reproduce bugs
- Provide relevant context (OS, Go version, email provider, etc.)

### Submitting Pull Requests

Before submitting a PR, please ensure:

1. **Changes are small and focused** — each PR should address a single concern or feature
2. **Code is thoroughly tested** — include unit tests, integration tests, or manual testing evidence
3. **Changes are reliable** — avoid experimental or untested code paths
4. **Commit messages are clear** — explain the "why" behind your changes

### AI-Generated Code

AI-assisted contributions are allowed. However, **you own the code you submit**. This means:

- You are responsible for understanding every change in your PR
- You must verify that the code works correctly and is production-ready
- Changes must meet the same standards as human-written code (small, reliable, tested)
- Low-quality or obviously untested AI output ("AI Slop") will be rejected
- Repeated submissions of AI Slop may result in being blocked from contributing

If you use AI tools to help with your contribution, that's fine — just make sure you've done the due diligence to ensure it's solid work.

### Code Quality

I prioritize:

- **Correctness** — code should work reliably for its intended purpose
- **Clarity** — code should be readable and maintainable
- **Testing** — changes should be covered by tests
- **Security** — especially important for email/authentication handling

### Questions?

Feel free to open an issue to discuss your idea before investing time in a PR.

Thank you for contributing!
